트러블슈팅
* 	공부한 내용
jwt 토큰을 이용한 로그인/회원가입 작성 방법


어려웠던 점, 해결방법
* 몽고디비 오류 -> certifi 패키지 설치, mongodb 주소 다시 한번 입력 확인
* ec2 연결 오류 -> 패키지 미설치
* 로그인, 회원가입시 토큰으로 클라이언트와 서버가 소통하는 방식을 처음에 이해하기 굉장히 어려웠다.
* 깃허브 ( 머지, 브랜치) : 서로 도와 강의와 구글링으로 해결
* DB로 데이터를 저장하는 과정중에 로컬ip주소 인식오류로 접근허용이 불가한 부분을 구글링을 통해 해결

해결 못한점
1.로그인에 따라 각자의 화면 구현
2. CSS class들이 겹쳐서 디자인을 통일하는데 많은 어려움이 있었고 불필요한 코드에 대한 정리작업도 하지 못했던 점. .
3. 몽고 db , ec2와 직접 연결 못함
4. 캘린더에 운동일지 적은 날짜 표시 및 날짜 선택 후 todolist페이지로 넘어간 후에 작성날짜 표기 구현안된점.
5.전체적인 구성
6.JWT토큰으로 http보안 연결 못한점, 개념은 이해했으나 코드로 녹이지 못함



* 아쉬운점
				
* 다른 사람들의 운동일지 공유
* 달력하고 일지 연결(날짜에 맞게 일지 연결)
* 로고, 디자인(버튼, 배경 등) 업그레이드
* 서버의 개인화
* jinja2를 적극적으로 활용하지 못한 점
* 처음 역할분배시 효율적으로 분배하지 못했던 점
* 서버와 클라이언트의 이해가 아직 부족했던 점
* 초반 API설계, 기능 구현 때 구현하고 싶은 기능이 너무 많아서 개인별 담당 페이지를 정하지 않고 하루를 날림.
* 디자인적인 문제, 아직 모르는 css, js 코드들을 구글링에서 하나씩찾아서 원하는 포지션에 넣는데 구현이 생각대로 되지 못함.

	



처음 역할 분배시 효율적으로 분배하지 못했던 점
역할을 분배를 잘해야 깃을 해도 부딪히지 않을텐데 처음에 분배가 명확히 잘이루어지지 않아 시간을 많이 날리게 되었고 초반에 협업에 어려움이 있어서 아쉬웠습니다.
그래서 중간에 
구현하고 싶은  기능이 많았는데 페이지 구현과 서버관리가 아직 미숙해서 잘 구현이 되지 않아서 아쉬웠다
로그인시 운동일지 다른사람과 공유, 달력과 페이지 연결 등 구현하고 싶은 기능이 많았지만 아직 서버와 클라이언트의 이해가 아직 부족한 점이 있어서 
코드 작성시 어려움이 있었다. 그리고 배운 코드들을 많이 응용하지 못해서 그런 점들이 아쉬웠습니다.
